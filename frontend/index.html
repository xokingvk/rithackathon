Frequently asked

What format should my CSV be in?
Columns: timestamp, voltage, current, temperature, cycle_count. The soc (state of charge %) column is optional. Download the template from the upload panel for an exact example.
Can I monitor a whole fleet?
Yes — the dashboard's fleet view aggregates SoH and RUL across every registered pack. Upload separate CSVs with different Battery IDs, or submit manual entries per pack. The fleet analytics table ranks by risk automatically.
How is remaining useful life calculated?
A gradient-boosted model trained on degradation trends projects the cycle at which SoH crosses your end-of-life threshold (default 70%). Without a trained .pkl file, a physics-informed heuristic using fade rate and temperature penalties provides a usable estimate.
How do I add my own XGBoost models?
Drop soh_model.pkl, rul_model.pkl, and charging_model.pkl into backend/app/ml/models/. Save with joblib.dump(model, path). Any estimator with .predict(X) works — restart the server and the new models are picked up automatically.
How do I enable the AI chatbot?
Add GROQ_API_KEY=gsk_... to backend/.env. Get a free key at console.groq.com. Without a key the chatbot still responds with helpful demo messages so the widget is always usable.
Can I use PostgreSQL instead of SQLite?
Yes — set DATABASE_URL=postgresql+psycopg2://user:pass@host:5432/voltaiq in your .env. No other changes needed. SQLite is the default for zero-setup local development.
